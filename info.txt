# **Project Title:**

**Emotion Detection from Text Using Machine Learning**

---

# **1. Dataset Used**

* **Source:** [Kaggle ‚Äì Text Dataset for Emotion Detection](https://www.kaggle.com/datasets/jarvis11/text-dataset-for-text-emotion-detection)
* **Description:** Contains **40,000 text entries** (tweets) labeled with emotions such as `neutral`, `happiness`, `worry`, `sadness`, `love`, `fun`, `surprise`, etc.
* **Columns in dataset:**

  1. `tweet_id` ‚Äì Unique ID of the tweet
  2. `sentiment` ‚Äì Label for emotion
  3. `content` ‚Äì Text of the tweet

---

# **2. Algorithm Used**

* **Text Preprocessing:** Clean the text (remove URLs, mentions, hashtags, punctuation, numbers, lowercase conversion).
* **Vectorization:** Convert text into numerical features using **TF-IDF Vectorizer** with unigrams, bigrams, and trigrams.
* **Machine Learning Model:**

  * **Logistic Regression** ‚Äì used for multi-class classification.
  * Trained to map textual patterns to corresponding emotions.

---

  * `pickle` ‚Äì for saving the trained model

# **5. Step-by-Step Procedure**

1. **Load Dataset:**

   * Load CSV and select necessary columns (`content`, `sentiment`).

2. **Simplify Labels:**

   * Kept **top 7 emotions** with sufficient data to avoid imbalanced classes.

3. **Balance Dataset:**

   * Used **resampling** so each emotion has roughly equal number of samples.

4. **Text Cleaning:**

   * Lowercase all text
   * Remove URLs, mentions, hashtags, numbers, punctuation
   * Strip extra spaces

5. **Split Dataset:**

   * **80% training**, **20% testing** using `train_test_split`

6. **TF-IDF Vectorization:**

   * Convert cleaned text into numerical features (max 15,000 features)
   * Include **unigrams, bigrams, trigrams** for better context capture

7. **Train Model:**

   * Logistic Regression with `max_iter=1000` and `C=2`

8. **Evaluate Model:**

   * Accuracy and **classification report**
   * Confusion matrix visualization

9. **Save Model and Vectorizer:**

   * Saved as `emotion_model.pkl` and `tfidf_vectorizer.pkl` for reuse

10. **Deploy Streamlit App (Optional):**

    * User inputs text
    * App predicts emotion using trained model
    * Display predicted emotion (optionally with emoji)

# **6. Problems Faced and Solutions**

| Problem                                                | Solution                                                                               |
| ------------------------------------------------------ | -------------------------------------------------------------------------------------- |
| Dataset imbalance ‚Äì many emotions had very few samples | Kept top 7 emotions and used **resampling** to balance data                            |
| Model accuracy was low (~32%)                          | Switched from Naive Bayes to **Logistic Regression** and used **TF-IDF with trigrams** |
